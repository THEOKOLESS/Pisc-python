Ex01 : Processus d'affichage d'une page statique.

Vue d'ensemble

Lorsqu'un utilisateur accède à une URL dans un projet Django, une série d'étapes précises se déroule, de la réception de la requête HTTP jusqu'à l'envoi de la réponse HTML au navigateur.

1. La requête HTTP arrive

Le navigateur envoie une requête HTTP GET vers le serveur Django (ex : GET /ex01/affichage). Le serveur de développement (manage.py runserver) reçoit cette requête et la transmet à Django.

2. Résolution de l'URL

Django consulte le fichier urls.py du projet (ROOT_URLCONF), qui redirige vers le urls.py de l'application concernée. Django parcourt les patterns URL jusqu'à trouver celui qui correspond au chemin demandé et identifie la vue associée.

3. Appel de la vue

Django appelle la fonction de vue correspondante en lui passant l'objet HttpRequest. La vue contient la logique applicative : elle peut récupérer des données, effectuer des calculs, etc. Dans le cas d'une page statique, elle se contente d'appeler render().

4. Chargement et rendu du template

La fonction render(request, 'template.html') demande au moteur de templates Django de localiser le fichier HTML correspondant (dans les dossiers templates/ des apps installées), puis de le rendre : les balises Django ({% block %}, {% include %}, variables) sont interprétées et remplacées par leur valeur finale.

5. La réponse HTTP

Django encapsule le HTML généré dans un objet HttpResponse avec le code de statut 200 OK et le renvoie au navigateur, qui l'affiche à l'utilisateur.

Schéma résumé

Navigateur
   │  GET /ex01/affichage
   ▼
Django (urls.py)
   │  pattern trouvé → vue affichage()
   ▼
Vue (views.py)
   │  render(request, 'ex01/affichage.html')
   ▼
Moteur de templates
   │  rendu HTML final
   ▼
HttpResponse (200 OK)
   │
   ▼
Navigateur affiche la page
