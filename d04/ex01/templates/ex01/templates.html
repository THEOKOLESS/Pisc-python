Ex01 : Moteur de template.

Le moteur de templates Django

Django intègre son propre moteur de templates : le DTL (Django Template Language). Il permet de générer du HTML dynamiquement en mélangeant du HTML statique avec des balises et variables Django. Le moteur reçoit un template et un contexte (dictionnaire de données), les combine, et produit un document HTML final.

Les deux syntaxes fondamentales sont :

  • {{ variable }} — affiche la valeur d'une variable
  • {% tag %} — exécute une logique (boucle, condition, bloc...)

Les blocs

Les blocs permettent l'héritage de templates. Un template parent définit des zones nommées avec {% block nom %}{% endblock %}. Un template enfant étend le parent avec {% extends "base.html" %} et redéfinit ces zones. 

{% verbatim %}{# base.html #}
<html>
<head>
    <title>{% block title %}{% endblock %}</title>
</head>
<body>
    {% block content %}{% endblock %}
</body>
</html>{% endverbatim %}
{% verbatim %}{# page.html #}
{% extends "base.html" %}

{% block title %}Ma page{% endblock %}

{% block content %}
    <p>Contenu de la page.</p>
{% endblock %}{% endverbatim %}

Le moteur fusionne les deux fichiers : le HTML du parent est conservé, et les blocs sont remplacés par ceux de l'enfant.

Les boucles for ... in

La balise {% for %} itère sur n'importe quel objet itérable passé dans le contexte (liste, queryset, dictionnaire...). 

{# views.py #}
def ma_vue(request):
    return render(request, 'page.html', {'fruits': ['pomme', 'banane', 'cerise']})
{% verbatim %}{# template #}
<ul>
{% for fruit in fruits %}
    <li>{{ fruit }}</li>
{% endfor %}
</ul>{% endverbatim %}

Des variables spéciales sont disponibles dans la boucle : forloop.counter (index à partir de 1), forloop.first, forloop.last. On peut gérer le cas d'une liste vide avec {% empty %}. 

{% verbatim %}{% for item in liste %}
    {{ forloop.counter }}. {{ item }}
{% empty %}
    <p>La liste est vide.</p>
{% endfor %}{% endverbatim %}

Les structures de contrôle if

La balise {% if %} évalue une condition et affiche le bloc correspondant. Elle supporte {% elif %} et {% else %}. 

{% verbatim %}{% if user.is_authenticated %}
    <p>Bonjour, {{ user.username }} !</p>
{% elif user.is_anonymous %}
    <p>Vous n'êtes pas connecté.</p>
{% else %}
    <p>Statut inconnu.</p>
{% endif %}{% endverbatim %}

Opérateurs disponibles : ==, !=, <, >, in, not in, and, or, not.

Affichage de variables passées par contexte

La vue passe un dictionnaire appelé contexte au template via render(). Chaque clé devient une variable accessible dans le template avec {{ clé }}. 

{# views.py #}
def ma_vue(request):
    context = {
        'titre': 'Bonjour',
        'score': 42,
        'user': {'nom': 'Alice', 'age': 30},
    }
    return render(request, 'page.html', context)
{% verbatim %}{# template #}
<h1>{{ titre }}</h1>
<p>Score : {{ score }}</p>
<p>Utilisateur : {{ user.nom }}, {{ user.age }} ans</p>{% endverbatim %}

La notation pointée {{ user.nom }} fonctionne pour les dictionnaires, les attributs d'objets et les indices de listes.
